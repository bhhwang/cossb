5 Features
6 ===============
7 1. IoT(Internet of Things) software framework for Robotic services.
8 2. Service components can be downloaded/installed/uninstalled/updated dynamically responding to what devices have services in same network environment. Thus, our goal is that any services can be changed dynamically with no user or supervisor intervention. (Except the initial setting up).
9 3. Fast by using C/C++. Lesser computational power might be required under the same condition. You can obtain more advantages for the device form-factor, budget and so on..
10 4. loose-coupled with other services by Service Broker. This gives advantages like a service re-configuration or re-combination)
11 5. Not only for High performance computing machine(like a PC), It is going for the cheap embedded hardware devices like Raspberry Pi, Odroid..

31 COSSB Usage
32 -------
33 1. Edit manifest.xml
34 
35 Manifest.xml file contained all information about running environments like below.
36 You should edit just <required> tags, then the COSSB components surrounding <required> tag will be loaded.
37 
38 ```
39 <?xml version="1.0" encoding="utf-8"?>
40 <manifest>
41  <!-- product information (IMPORTANT section) -->
42  <product>
43  <serial>NSGW00000000FFFFFFFFFFFF</serial>
44  <manufacturer>Nsynapse Inc.</manufacturer>
45  <license>cdefabcd-cdefabcd-cdefabcd-cdefabcd</license>
46  <hostname>cossbgw.local</hostname>
47  </product>
48  <!-- accessible repository server address -->
49  <repository>
50  <uri></uri>
51  </repository>
52  <!-- required component cannot be stopped by user -->
53  <!-- required library for advanced user or our support -->
54  <required type="component" for="service">compzeroconf</required>
55  <required type="component" for="service">messagepub</required>
56  <required type="library" for="log">libcblog.so</required>
57  <!-- do not edit this path -->
58  <property>
59  <path name="home">./cossb_home/</path>
60  <path name="include">./include/</path>
61  <path name="component">./component/</path>
62  <path name="library">./library/</path>
63  </property>
64 
65 </manifest>
66 ```
67 
68 2. Just run with manifest
69 
70 COSSB will find and load manifest.xml file at first when gives 'run' option.
71 
72 ```
73 $ ./cossb --run
74 ```

76 COSSB Utility Usage
77 -------
78 1. createcomp (Not support yet completely)
79 
80 This utility generates empty component class and its default profile into the own directory. Target directory will be created as 'CURRENT_DIRECTORY/COMPONENT_PATH/YOUR_CLASSNAME'. Note that this utility requires root permission to create directories.
81 ```
82 $ sudo ./cossb --utility createcomp <your component class name>
83 ```
